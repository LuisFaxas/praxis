# Commands

Operator command delivery. AI writes commands here; the admin (human) executes them.

## Pipeline Position

Commands are an execution-stage artifact, supporting work orders during the build phase.

## Structure

- `active/` -- Current command sets organized by topic (each topic gets its own subfolder)
- `executed/` -- Completed command sets

```
commands/
├── active/
│   └── {number}_{date}_{TOPIC}/    # Topic subfolder
│       ├── MASTER_COMMAND_DOC.md   # Steps with numbered commands
│       └── (supporting scripts)    # Scripts, configs, etc.
└── executed/
    └── (completed topic subfolders)
```

## Rules

- Each set of related commands gets its own dated topic subfolder in `active/`
- The master command doc has numbered steps the admin executes
- **AI never pastes multiline commands in chat** -- write them here and reference the doc path + step number
- When all commands in a topic are complete, move the entire subfolder to `executed/`

## Example Topic Subfolder

```
03_2026-02-20_SSH_BACKEND_SETUP/
├── SSH_SETUP_MASTER.md         # "Run Step 1, then Step 2..."
├── install_deps.sh             # Script referenced by Step 1
└── nginx.conf                  # Config referenced by Step 3
```

## Naming Convention

Topic subfolder: `{number}_{YYYY-MM-DD}_{TOPIC}/`
Files within: descriptive names related to the topic
