# Master Plan

High-level project plans that go through a two-stage review pipeline.

## Purpose

Master plans define the overall project strategy, phases, and deliverables. They follow the same draft/published separation as reports: AI writes to `draft/`, admin promotes to `approved/`.

## Structure

- `draft/` -- Working plans being developed by AI agents
- `approved/` -- Finalized plans promoted by admin after review

## Rules

- AI writes plans to `draft/` only
- Only the admin moves plans from `draft/` to `approved/`
- Approved plans are decomposed into work orders for execution

## Naming Convention

`{number}_{YYYY-MM-DD}_{DESCRIPTION}.md`
