# Planning

Master plans and architectural decisions.

## Pipeline Position

```
Research  -->  Planning  -->  Execution  -->  Reports
(gather)      (decide)       (build)        (communicate)
                ^^^
             YOU ARE HERE
```

## Purpose

The planning folder organizes all high-level project planning artifacts. Plans go through a draft/approved pipeline. Once approved, they are decomposed into work orders in `dev/work-orders/`.

## Structure

- `master-plan/` -- High-level project plans (draft --> approved pipeline)

## Workflow

1. **Discovery:** AI reads the codebase and generates a project review
2. **Draft Plan:** AI writes master plan to `master-plan/draft/`
3. **Review:** Admin reviews, requests changes, iterates
4. **Approval:** Admin moves finalized plan to `master-plan/approved/`
5. **Decomposition:** Plan is broken into work orders in `dev/work-orders/`
6. **Execution:** AI agents execute WOs, creating audit entries and command docs as they work
7. **Completion:** Completed WOs move to `work-orders/executed/`

## Naming Convention

`{number}_{YYYY-MM-DD}_{DESCRIPTION}.{ext}`
