# Audit

AI-driven codebase and architecture assessments that feed into work order creation.

## Purpose

The audit folder is where AI agents perform comprehensive project assessments. The admin sends an AI to audit the codebase, and the AI generates detailed reports that identify issues, evaluate risks, and produce findings that are later analyzed and converted into new work orders.

## Structure

- `current/` -- Active audit reports for the current phase of work
- `legacy/` -- Archived audit reports from completed phases (admin moves here when a phase completes)

## Audit Types

| Type | Purpose |
|------|---------|
| **Architecture Audit** | Deep assessment of system design with per-component KEEP/CHANGE/REMOVE verdicts |
| **Risk Register** | Systematic risk identification with impact/likelihood matrix, detection signals, and mitigation plans |
| **Readiness Audit** | Phase completion gate check -- go/no-go recommendation before proceeding |
| **Drift Check** | Detect when documentation contradicts the Source of Truth |
| **Design Audit** | Visual review of UI/UX via Puppeteer screenshots |
| **Incident Post-Mortem** | Root cause analysis after failures with corrective actions |

## Workflow

1. Admin creates a WO requesting an audit
2. AI reads the codebase, configs, and documentation
3. AI generates an audit report with verdicts and findings
4. (Optional) A second AI reviews the audit for multi-agent verification
5. Audit findings are decomposed into new work orders

## Naming Convention

`{number}_{YYYY-MM-DD}_{DESCRIPTION}.md`

- Number is sequential and chronological
- Date is the creation date in ISO format
- Description is uppercase, underscore-separated
