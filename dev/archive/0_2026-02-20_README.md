# Archive

Historical records preserved for traceability. When legacy documents are retired, they move here with a manifest explaining what was archived and why.

## Structure

Each archive operation creates a dated batch:

```
archive/
└── {YYYY-MM-DD}_{description}/
    ├── 0_{date}_ARCHIVE_MANIFEST.md    # Maps original paths → archive paths + rationale
    └── (archived files organized by category)
```

## Rules

- Every archive batch includes a manifest documenting what was moved and why
- Archive operations are admin-initiated, not automatic
- Archived files retain their original names for traceability
- The manifest should include the canonical replacement (if any) for each archived file

## When to Archive

- Root markdown sprawl cleanup (legacy docs that predate the dev/ methodology)
- Superseded plans (old master plans replaced by new versions)
- Completed incident records (post-mortems, recovery docs)
- Deprecated documentation
