# Work Orders

Actionable tasks decomposed from the master plan. Work orders are the execution stage of the development lifecycle.

## Pipeline Position

```
Research  →  Planning  →  Execution  →  Reports
(gather)    (decide)     (build)       (communicate)
                           ^^^
                        YOU ARE HERE
```

## Structure

### Solo Mode (single agent)

Work orders live directly in this folder. No agent prefix needed.

```
work-orders/
├── (active WOs here)
└── _executed/
```

### Triangle Mode (multi-agent)

When multiple AI agents are active, work orders are routed via agent-specific subfolders:

```
work-orders/
├── wo_claude/
│   ├── 10_delivery_academy/        # Lane: active WOs
│   ├── 20_delivery_site_core/      # Lane: active WOs
│   └── _executed/
│       ├── 10_delivery_academy/    # Lane: completed WOs
│       └── 20_delivery_site_core/
├── wo_codex/
│   └── _executed/
└── wo_gemini/
    └── _executed/
```

Triangle mode is activated when `CODEX_INIT.md` and/or `GEMINI_INIT.md` exist in `dev/init/`.

### WO Lanes (optional)

Lanes organize WOs into typed subproject scopes: `{nn}_{type}_{scope}` (e.g., `10_delivery_academy`). Lane types: `delivery`, `program`, `lab`, `ops`. Program and lab lanes have relaxed validation (no Acceptance Criteria or Status required).

### Patch Work Orders

Extend a completed parent WO using `_P{NN}` suffix: `5_2026-02-22_FIX_BUG_P01.md`. Patches include parent metadata (`**Parent WO:**`, `**Patch:**`, `**Sequence Key:**`).

### N/A Criteria

Mark inapplicable criteria as: `- [ ] ~~text~~ N/A — reason`. Requires a reason, max 3 per WO, prefer rewriting in active WOs.

## Rules

- Active WOs live in the folder root (or agent subfolder / lane in Triangle mode)
- Completed WOs move to `_executed/` (or `_executed/{lane}/` for lane WOs) — items stay active until fully done
- Only create WOs for actionable, near-term work — long-term items stay in the master plan
- Each WO has clear acceptance criteria so "done" is unambiguous
- Legacy `executed/` naming is supported but `_executed/` is preferred for new projects

## Naming Convention

`{number}_{YYYY-MM-DD}_{DESCRIPTION}.md`
`{number}_{YYYY-MM-DD}_{DESCRIPTION}_P{NN}.md` (patch WOs)
