# Design -- Audit

Before/after screenshots and visual review artifacts from design audits.

## Purpose

When AI performs a design audit (comparing implementation against brand guidelines), it captures screenshots here. These provide visual evidence for audit reports and help track UI drift over time.

## Workflow

1. AI captures current UI state via Puppeteer MCP into `screenshots/`
2. AI compares captures against `design/language/` specifications
3. Findings are documented in a design audit report in `dev/audit/current/`

## Naming Convention

`{number}_{YYYY-MM-DD}_{DESCRIPTION}.{ext}`
