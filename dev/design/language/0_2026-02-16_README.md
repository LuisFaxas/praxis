# Design -- Language

Design language assets: tokens, brand guidelines, and Storybook methodology docs.

## Purpose

"Design language" is a broad concept covering everything that defines the visual identity of a project. This folder serves two use cases depending on the project type:

**Non-frontend projects** (infrastructure, CLI, API-only):
- `tokens.json` — Machine-readable design tokens (colors, spacing, typography)
- `brand-guidelines.md` — Human-readable brand rules
- AI agents read these when generating HTML reports

**Frontend projects with Storybook:**
- Storybook IS the design language. Design tokens live in `tailwind.config.ts` + `globals.css`
- This folder stores Storybook methodology docs: story conventions, deployment notes, component organization
- `tokens.json` may be absent since Storybook renders tokens as living documentation

## Expected Files

| File | Used By | Purpose |
|------|---------|---------|
| `tokens.json` | Non-frontend projects | Machine-readable design tokens (colors, spacing, radius, shadow, typography) |
| `brand-guidelines.md` | All projects | Human-readable brand rules (tone, logo usage, color context, do's and don'ts) |
| `storybook-conventions.md` | Frontend projects | Story naming, structure, args patterns |
| `storybook-deployment.md` | Frontend projects | Deployment notes (Vercel, Chromatic, etc.) |

## Important Note

The actual `.storybook/` config directory (main.ts, preview.tsx) must live at the **project root** per Storybook tooling convention. This folder is for methodology docs only, not tooling config.

## Integration

- **HTML Reports:** AI reads `tokens.json` to apply correct CSS custom properties
- **Storybook:** Tokens feed into CSS variables for component library theming (or live in tailwind.config.ts)
- **Design Audits:** AI compares implementation against token values
