# Design

Brand assets, design tokens, and visual audit artifacts that maintain design consistency across the project.

## Purpose

The design folder centralizes all visual design resources. AI agents reference these assets when generating HTML reports, reviewing UI components, or performing design audits. This ensures consistent brand presentation without hardcoding values in templates.

## Structure

- `audit/` -- Visual design review artifacts (before/after screenshots)
- `language/` -- Design language assets: tokens, brand guidelines, and Storybook methodology docs
- `resources/` -- Static assets (icons, fonts, logos)

## Design Language Strategy

The `language/` folder adapts to the project type:

**Non-frontend projects (infrastructure, CLI, API-only):**
- `tokens.json` with design tokens used by HTML report generation agents
- `brand-guidelines.md` with human-readable brand rules

**Frontend projects with Storybook:**
- Storybook IS the design language. Design tokens live in `tailwind.config.ts` + `globals.css`
- `language/` stores Storybook methodology docs (story conventions, deployment notes)
- Actual `.storybook/` config lives at the project root per tooling convention

## Workflow

1. **Brand setup:** Populate `language/tokens.json` with project design tokens and `brand-guidelines.md` with human-readable brand rules
2. **Report generation:** AI reads `language/tokens.json` when generating HTML reports to use correct colors, fonts, and spacing
3. **Design audits:** AI captures screenshots into `audit/screenshots/`, then compares against specifications
4. **Storybook integration (frontend):** Use `storybook/` for methodology docs; actual `.storybook/` config lives at project root per tooling convention
